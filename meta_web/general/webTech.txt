JS
  high level programming lang has to be converted to binary code so that a CPU can work with it
  used to to add mouseover effects and add interactivity to web pages or websites
  Java and JavaScript are in no way related
  but since the Java lang was so popular at the time, the name was changed from LiveScript to JavaScript
  is not a completely separate, stand-alone entity, exists as an implementation known as a JS engine
  Traditionally, the only environment in which it was possible to run a JavaScript engine, was the browser
  JS engine was just another building block of the browser
  was there to help a browser accomplish its users' goal of utilizing the internet for work, research, play ..
  JS engine itself comes with different ways to interact with various other parts of the browser (APIs)
  Thus, the code that you write in the JS allows you to:
    - Interact with the JS engine inside of the browser
    - Interact with other browser func exists outside of the JS engine, but is still inside the browser
  this all changed in 2009, when Node.js was built by a way to use a JS engine as a stand-alone entity
  Finally, it was possible to use JS outside of the browser, as a sep program on cmd or as a server-side env
ECMAScript
  In 1996 Netscape made a deal with the organization known as ECMA to draft the specification of the JS lang
  ECMA-262 is a standard that specifies how the JS lang should work
  There have been 12 ECMA-262 updates - the first one was in 1997

React
  is a JS library relies on DOM (lib not framework)
  a lib for building composable UIs present data that changes over time by breaking them into comps
  when your comp is first init, render method is called
  generating a lightweight representation of your view
  (a string of markup is produced and injected into the doc)
  When your data changes, the render method is called again
  in order to perform updates as efficiently as possible
  we diff the return val from the prev call to render with the new and gen a min set of changes to DOM
  data returned from render is neither string nor DOM node

React Fiber Arch
  allows React to incrementally render the web page
  instead of immediately updating DOM with all virtual DOM changes, React can spread the update over time
  React can optimize when and where updates occur to DOM to significantly improve app performance and responsiveness to user input
  it's as a (priority sys):
    highest priority changes, the elems visible to the user (updated first)
    lower priority changes, the elems not currently displayed (updated later)
